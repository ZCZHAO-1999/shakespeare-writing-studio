# Editorial QA

Use this before delivery.

## Content

- [ ] The central claim is clear in one sentence.
- [ ] The article has one main spine, not several competing arguments.
- [ ] All facts, dates, names, and numbers are from the draft or marked `[TK]`.
- [ ] Claims that need sourcing are softened or flagged.
- [ ] The piece has examples, not only assertions.
- [ ] If external data is used, every number has a source and date.
- [ ] If no data is used, that choice is intentional and appropriate to the format.

## Voice

- [ ] No generic AI opening.
- [ ] No "not only X, but also Y" scaffolding.
- [ ] No repeated paragraph rhythm for more than three paragraphs.
- [ ] No inflated business jargon.
- [ ] The writer's point of view is visible.
- [ ] The ending lands instead of summarizing.

## Platform

- [ ] Title is specific and not clickbait.
- [ ] Subtitle/deck gives context without repeating the title.
- [ ] Sections are scannable.
- [ ] Paragraphs are short enough for web reading.
- [ ] Pull quotes, if used, are actually quotable.
- [ ] Tags match the article, not the hoped-for audience.
- [ ] Charts or visuals, if used, clarify the article rather than decorating it.
- [ ] A statistics-heavy piece has the right visual type for the data.

## Publishing Note

Include:

- primary title
- alternate titles
- subtitle
- preview description
- social teaser
- suggested tags
- unresolved `[TK]` items
- AI-assistance disclosure note if needed
