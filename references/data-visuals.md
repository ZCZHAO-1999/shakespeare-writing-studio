# Data Visuals Skill

Use this module to decide whether data should become a chart, data card, timeline, map, word cloud, or stay as prose.

## 1. First Question

Does the visual help the reader understand the argument faster?

If no, do not make a chart.

## 2. Chart Decision Matrix

| Reader Needs To See | Use | Avoid |
|---|---|---|
| Change over time | line chart | bar chart with too many years |
| Ranking / comparison | horizontal bar chart | pie chart |
| Before vs after | slope chart | clustered bars |
| Relationship | scatter plot | fake correlation |
| Distribution | histogram / box plot | average-only chart |
| Parts of a whole | stacked bar | pie chart unless very simple |
| Geography | map | map without meaningful location pattern |
| Process over time | timeline | dense paragraph |
| Repeated categories | heatmap | long table |
| Language / themes | word cloud or term frequency bars | decorative word cloud with no insight |
| One key number | data card | full chart |
| Many small comparisons | small multiples | overloaded single chart |

## 3. Creative Statistical Graphics

Simple charts are the floor, not the ceiling.

For research-led pieces, consider one hero visual that carries the argument:

- system map: shows actors, bottlenecks, and flows
- annotated scatter: places companies, countries, models, or policies in a meaningful coordinate system
- calendar heatmap: shows rhythm, bursts, seasonality, or adoption cycles
- radial timeline: shows diffusion, cascading events, or network effects
- matrix poster: combines source notes, mini charts, and interpretation
- bubble constellation: shows scale, influence, or concentration
- ranked slope wall: shows movement between two periods or two groups

The hero visual must still be source-bound. Beauty does not excuse vague data.

Use a creative graphic when:

- the article has several related numbers
- the reader needs to see a system, not one metric
- a normal line chart would under-explain the argument
- the visual can carry one clear thesis

Avoid a creative graphic when:

- it hides uncertainty
- the labels become unreadable
- the design needs invented data
- the visual becomes the article's argument without textual support

## 4. Word Clouds

Use word clouds sparingly.

Good for:

- showing repeated themes in interviews
- comparing language across documents
- exploratory appendix visuals

Avoid when:

- exact comparison matters
- the source text is too small
- common words dominate
- the visual would look decorative

Prefer term-frequency bar charts when the reader needs precision.

## 5. Visual Density Rules

Default article:

- 0–2 visuals
- mostly prose

Analytical article:

- 1–4 visuals
- each visual supports a section

Research-led article:

- 3–7 visuals
- include source notes and caveats
- include one hero statistical graphic when the piece has a system-level argument

Memo:

- charts only when they support a decision

## 6. Chart Quality Gate

Every chart must have:

- clear title
- source line
- date or data range
- labeled axes
- readable labels
- one takeaway
- no unsupported numbers
- for hero graphics, one visible thesis line

Reject:

- chart with no source
- chart that repeats the paragraph
- tiny labels
- decorative chart with no decision value
- too many colors
- 3D chart
- pie chart with more than 3 slices

## 7. Matching Visual To Tone

Medium/Substack:

- clean chart
- one sentence takeaway
- minimal annotation

Executive memo:

- tighter chart
- source and caveat close by
- decision implication

Public essay:

- fewer charts
- more interpretation

Research article:

- chart sequence can carry the argument
